# 해빛스쿨 보상 마켓 서비스 소개서

## 1. 서비스 개요

해빛스쿨(Habit School)은 식단, 운동, 수면, 마음건강 같은 일상 건강 활동을 기록하고
지속적인 건강 습관 형성을 돕는 모바일 기반 서비스입니다.

사용자는 앱 안에서 건강 활동을 수행하며 포인트를 적립하고,
적립한 포인트를 `해빛 마켓`에서 모바일 쿠폰으로 교환할 수 있습니다.

현재 쿠폰 교환은 `앱 내 포인트 차감` 기반으로 운영하며,
`HBT`는 자산 탭의 별도 시스템으로 유지하고 쿠폰 발급 흐름과는 분리합니다.

## 2. 기프티쇼 API 도입 목적

기프티쇼 비즈 API는 해빛스쿨의 보상 마켓 기능 구현을 위해 사용합니다.

- 회원이 포인트로 원하는 리워드 상품을 선택하면
  해빛스쿨 서버가 Giftishow API로 쿠폰 발급을 요청합니다.
- 발급된 쿠폰은 문자 재판매가 아니라
  해빛스쿨 앱 내 `쿠폰 보관함`에서 바코드 이미지 또는 PIN으로 확인하도록 설계합니다.
- 운영자는 관제탑에서 발급 상태, 실패 건, 수동 재확인 내역을 관리합니다.

## 3. 현재 운영 정책

- 쿠폰 교환 자산: 앱 내 포인트
- 쿠폰 수령 방식: 앱 보관함 우선 노출
- 연락처 사용 범위: Giftishow 발급 요건 및 예외 대응 목적의 최소 범위
- HBT 처리 방식: 자산 탭 별도 운영, 현재 쿠폰 교환과 분리

## 4. 주요 이용 흐름

1. 회원이 해빛스쿨 앱에서 건강 활동을 수행합니다.
2. 활동 결과에 따라 앱 내 포인트가 적립됩니다.
3. 회원이 `해빛 마켓`에서 원하는 쿠폰 상품을 선택합니다.
4. 서버가 포인트 차감을 확인한 뒤 Giftishow API로 쿠폰 발급을 요청합니다.
5. 발급된 쿠폰은 앱 내 `쿠폰 보관함`에 저장되고, 사용자는 앱에서 직접 확인합니다.

## 5. 초기 운영 상품

- 메가MGC커피 `(ICE)아메리카노 모바일쿠폰`, 60일, 2,000P 교환
- 빽다방 `아메리카노(ICED) 모바일쿠폰`, 60일, 2,000P 교환

초기에는 소액 음료형 보상부터 시작해
건강 습관 보상 경험을 가볍게 검증하는 방향으로 운영합니다.

## 6. 화면 및 자산 정책

- 상품 이미지는 Giftishow 비즈 상품 원본 이미지를 사용합니다.
- 브랜드 로고는 운영자가 승인한 실제 로고 자산을 앱에 self-host 방식으로 반영합니다.
- 쿠폰 보관함에서는 Giftishow 응답 이미지가 있으면 우선 노출하고,
  없으면 상품 이미지를 fallback으로 사용합니다.

## 7. 운영 및 예외 처리

- 쿠폰 발급은 사용자 요청 시점에만 수행합니다.
- 발급 상태는 `pending_issue`, `issued`, `failed_manual_review`, `cancelled`로 관리합니다.
- 장애나 응답 누락이 발생하면 관제탑에서 수동 재확인 및 후속 처리를 진행합니다.
- 발급 이력, 포인트 차감, 예외 처리 내역은 서버 원장 기준으로 관리합니다.

## 8. 기대 효과

- 건강 활동 보상이 실제 생활 혜택으로 이어집니다.
- 포인트 적립과 리워드 교환이 앱 안에서 자연스럽게 연결됩니다.
- 쿠폰 확인과 운영 관리를 앱 보관함과 관제탑으로 분리해 사용자 경험과 운영 안정성을 함께 확보합니다.
