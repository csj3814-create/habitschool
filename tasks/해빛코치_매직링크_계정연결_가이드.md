## 목표

기존 방식은 사용자가

1. 앱에서 연결 코드 생성
2. 코드를 복사
3. 카카오톡에서 `!등록 코드` 입력

까지 해야 했습니다.

새 방식은 훨씬 단순합니다.

1. 카카오 해빛코치 1:1 채팅에서 `!연결`
2. 챗봇이 `앱에서 연결하기` 버튼 전송
3. 버튼을 누르면 해빛스쿨 앱이 열림
4. 앱에서 현재 로그인된 계정으로 연결 완료

즉, **복사/붙여넣기 없이 버튼 한 번으로 연결**하는 것이 목표입니다.

## 챗봇 쪽에서 이미 준비된 것

해빛코치 서버에는 아래가 이미 구현되어 있습니다.

- `!연결` 명령
- 10분짜리 연결 토큰 발급
- 앱 열기 링크 생성
- 앱이 연결을 완료할 수 있는 API

1. 토큰 조회

`GET https://habitchatbot.onrender.com/api/chatbot-connect/:token`

응답 예시:

```json
  "displayName": "최석수",
  "platform": "kakao",
  "expiresAt": "2026-04-05T02:10:00.000Z",
  "status": "pending"
```

2. 연결 완료

`POST https://habitchatbot.onrender.com/api/chatbot-connect/complete`

헤더:

```http

Content-Type: application/json
```

바디:

성공 응답 예시:

## 앱에서 해야 할 일

핵심은 4단계입니다.

1. URL의 `chatbotConnectToken` 읽기
2. 토큰 정보 조회
3. 사용자에게 연결 확인 UI 보여주기
4. Firebase ID 토큰으로 완료 API 호출

---

## 1. URL 파라미터 읽기

챗봇은 아래 형식으로 앱을 엽니다.

```text
https://habitschool.web.app/?chatbotConnectToken=TOKEN값#profile
```

앱 시작 시 아래를 읽으면 됩니다.

```js
function getChatbotConnectTokenFromUrl() {
    const params = new URLSearchParams(window.location.search);
    return params.get('chatbotConnectToken') || '';
}
```

중요:
- 토큰은 `window.location.search`에 있습니다.
- `#profile`은 프로필 탭으로 이동시키기 위한 힌트로 사용하면 됩니다.

---

## 2. 토큰 조회 API 호출

앱이 열리면 토큰이 있나 먼저 보고, 있으면 서버에서 유효성 확인을 합니다.

```js
async function fetchChatbotConnectTokenInfo(token) {
    const response = await fetch(`https://habitchatbot.onrender.com/api/chatbot-connect/${encodeURIComponent(token)}`);
    const data = await response.json().catch(() => ({}));

    if (!response.ok || !data.ok) {
        throw new Error(data.code || 'invalid_token');
    }

    return data;
}
```

이 단계의 목적:
- 만료 토큰인지 확인
- 어떤 카카오 계정과 연결될지 표시

---

## 3. 연결 확인 UI

권장 UX:

- 프로필 탭 진입 시 토큰이 있으면 모달 노출
- 문구:
  - `카카오 계정 "최석수"와 연결할까요?`
  - `현재 로그인된 해빛스쿨 계정과 연결됩니다.`
- 버튼:
  - `연결하기`
  - `취소`

권장 상태 처리:

- 만료됨: `연결 링크가 만료되었어요. 카카오 1:1 채팅에서 !연결을 다시 입력해 주세요.`
- 이미 완료됨: `이미 연결된 링크예요.`
- 성공: `해빛코치 연결이 완료되었어요.`

## 4. 연결 완료 API 호출

사용자가 `연결하기`를 누르면 현재 로그인된 Firebase 사용자 ID 토큰을 보내야 합니다.

```js
async function completeChatbotConnect(token) {
    const user = auth.currentUser;
    if (!user) {
        throw new Error('login_required');
    }

    const idToken = await user.getIdToken();

    const response = await fetch('https://habitchatbot.onrender.com/api/chatbot-connect/complete', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
            'Authorization': `Bearer ${idToken}`
        },
        body: JSON.stringify({ token })
    });

    const data = await response.json().catch(() => ({}));

    if (!response.ok || !data.ok) {
        throw new Error(data.code || 'connect_failed');
    }

    return data;
}
```

중요:
- 서버가 Firebase ID 토큰을 검증합니다.
- 앱이 임의 UID를 보내는 방식이 아니라, **현재 로그인 사용자를 서버가 직접 검증**하는 구조입니다.

## 권장 앱 흐름

```js
async function maybeHandleChatbotConnect() {
    const token = getChatbotConnectTokenFromUrl();
    if (!token) return;

    try {
        const info = await fetchChatbotConnectTokenInfo(token);

        openChatbotConnectModal({
            displayName: info.displayName,
            expiresAt: info.expiresAt,
            onConfirm: async () => {
                const result = await completeChatbotConnect(token);
                showToast(`해빛코치 연결 완료: ${result.kakaoDisplayName}`);
                clearChatbotConnectTokenFromUrl();
                closeChatbotConnectModal();
            },
            onCancel: () => {
                clearChatbotConnectTokenFromUrl();
                closeChatbotConnectModal();
            }
        });
    } catch (error) {
        showToast('연결 링크가 만료되었거나 사용할 수 없어요. 카카오톡에서 !연결을 다시 시도해 주세요.');
        clearChatbotConnectTokenFromUrl();
    }
}
```

## URL 정리

연결 성공/취소/실패 후에는 URL에서 토큰을 지우는 걸 권장합니다.

```js
function clearChatbotConnectTokenFromUrl() {
    const url = new URL(window.location.href);
    url.searchParams.delete('chatbotConnectToken');
    window.history.replaceState({}, '', url.toString());
}
```

이걸 해두면:
- 새로고침 시 다시 연결 모달이 뜨지 않음
- 토큰 URL이 남지 않음

## 에러 코드 처리 기준

앱에서 아래처럼 처리하면 충분합니다.

- `expired`
  - 링크 만료
  - 안내: `카카오톡 1:1 채팅에서 !연결을 다시 입력해 주세요.`

- `unauthorized`
  - 로그인 만료 또는 잘못된 ID 토큰
  - 안내: `앱에 다시 로그인한 뒤 다시 시도해 주세요.`

- `already_used`
  - 이미 사용한 링크
  - 안내: `이미 사용된 연결 링크예요.`

- `missing_auth`
  - 앱이 ID 토큰 없이 요청

- `missing_token`
  - 앱이 토큰 없이 완료 API 호출

## 기존 `!등록 코드`와의 관계

기존 수동 방식은 바로 제거하지 않아도 됩니다.

권장 운영:
- 기본 연결: `!연결`
- 예외 fallback: `!등록 코드`

즉, 앱 UI에서는 더 이상 연결 코드 생성을 메인으로 노출하지 않고,
필요한 경우에만 보조 옵션으로 남기는 것이 좋습니다.
