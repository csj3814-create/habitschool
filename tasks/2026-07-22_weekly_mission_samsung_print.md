# 주간 미션 기본 선택 · 삼성 파일 선택 안내 · 30일 결과지 A4 2-up

## 목표

- 주간 미션을 새로 정하는 화면을 열면 식단·운동·마음 세 분류를 모두 기본 선택한다.
- 삼성 인터넷의 Android 작업 선택 화면에서 사용자가 올바른 `파일` 항목을 고르도록 정확히 안내한다.
- 30일 종합 결과지를 A4 한 장에 위·아래 두 구역으로 배치해 반으로 나눠 사용할 수 있게 한다.

## 원칙

- 기존 활성 미션과 저장된 선택·난이도는 덮어쓰지 않는다.
- 웹/PWA가 Android 시스템 선택기의 앱이나 항목을 강제로 선택할 수 있는지 먼저 확인하고, 불가능한 동작은 구현하지 않는다.
- 화면용 결과지 구성과 데이터 계산은 변경하지 않고 인쇄 전용 레이아웃만 추가한다.
- 기존 작업 트리의 `tasks/lessons.md` 수정과 미추적 문서 두 개를 보존한다.
- 사용자 확인 없이 커밋·푸시·배포하지 않는다.

## 조사 결과

- 주간 미션 선택기는 `weeklyMissionData`가 이번 주 활성 상태가 아닐 때만 노출된다. 활성 미션과 레거시 미션은 별도 진행 화면으로 렌더되므로 신규 선택기의 체크 기본값만 바꾸면 저장 데이터는 변하지 않는다.
- 식단·운동·마음 미디어는 네이티브 `<input type="file">` 경로를 사용한다. `accept`는 허용 형식 힌트이고 `capture`는 카메라 캡처 방향 힌트일 뿐, Android 작업 선택기의 특정 앱이나 provider를 지정하지 못한다.
- 따라서 삼성 인터넷에서 세 번째 `파일`을 자동 선택하거나 기본 포커스로 여는 것은 일반 웹/PWA에서 불가능하다. 기존 네이티브 입력을 유지하고 삼성 인터넷에서만 선택 버튼 근처에 지속 안내를 표시하는 것이 안전하다.
- 기존 인쇄 CSS는 화면의 `body` flex를 해제하지 않아 앱 화면과 결과지 모달이 A4 폭을 나눠 가졌다. 긴 결과지와 canvas 차트도 단순 복제만으로는 안정적인 2-up이 되지 않는다.
- 인쇄 직전에 결과지 구간을 둘로 분리하고 canvas를 이미지로 스냅샷한 A4 전용 시트를 생성한 뒤, 각 절반의 높이에 맞게 축소하고 인쇄가 끝나면 제거한다.

## 남은 문제·실기기 확인

- 삼성 인터넷의 Android 시스템 `작업 선택` 화면에서 세 번째 항목의 표시명이 실제로 `파일`인지, 안내 위치와 흐름이 자연스러운지는 staging 배포 후 삼성 실기기에서 최종 확인해야 한다.
- 웹/PWA에서는 시스템 선택기의 세 번째 `파일`을 자동 선택할 수 없다. 이 동작이 반드시 필요하면 Android 네이티브 래퍼가 파일 선택 요청을 중개하는 별도 제품 범위가 필요하다.
- 로그인된 실제 회원의 활성 미션 데이터 불변과 인쇄 미리보기 `1/1` 표시는 자동 테스트와 로컬 대표 PDF로 검증했으며, staging 운영 흐름에서 한 번 더 확인하는 것이 좋다.
- 커밋·푸시·배포는 수행하지 않았다.
