# 30일 결과지 원본 1·2페이지 A4 한 장 인쇄 교정

## 사용자 교정

- 기존 결과지 첫 페이지가 정상적으로 보여야 한다.
- 기존 1페이지와 2페이지가 A4 용지 한 장에 함께 들어가야 한다.
- 화면을 스크롤하던 위치나 결과지 하단 버튼이 인쇄물에 섞이면 안 된다.

## 확인된 원인

- Android 인쇄 미리보기에는 인쇄 전용 시트에 존재하지 않는 `A4 한 장에 인쇄`, `닫기` 버튼이 나타났다.
- 이는 `window.print()` 직후 Android/Samsung 인쇄 흐름에서 `afterprint`가 실제 스풀 캡처보다 먼저 발생해 `remove30DayReportPrintSheet()`가 인쇄 전용 시트와 `report-printing` 상태를 제거했음을 뜻한다.
- 상태가 제거된 뒤 인쇄 엔진이 화면용 고정 모달을 현재 스크롤 위치부터 다시 캡처하면서 첫 페이지 상단 누락, 섹션 중복, 총 3페이지가 발생했다.
- 로컬 정적 PDF 검증은 인쇄 전용 시트 자체만 검증해 이 모바일 수명주기 문제를 놓쳤다.

## 최소 수정 설계

- 결과지 데이터 계산과 화면용 모달은 변경하지 않는다.
- 기존 인쇄 전용 시트의 1면(헤더·요약·카테고리·포인트)과 2면(카테고리 추이·건강지표·히트맵) 구성을 유지한다.
- `afterprint`에서 즉시 제거하지 않는다. 인쇄 전용 시트는 화면에서는 계속 숨겨져 있으므로 결과지 모달을 닫거나 다음 인쇄를 준비할 때 정리한다.
- 닫기 버튼을 전용 `close30DayReport()`로 연결해 모달과 인쇄 시트를 함께 정리한다.
- A4 한 장의 두 면을 명시적으로 유지하고, 원본 첫 면의 헤더와 섹션 순서가 바뀌지 않는지 회귀 테스트한다.
- Android에서 인쇄 함수가 반환된 직후에도 `report-printing` 상태와 인쇄 전용 시트가 남아 있는지 테스트한다.

## 예상 수정 파일

- `js/app-core.js`: 조기 `afterprint` 정리 제거, 결과지 닫기 시 안전한 정리 함수 추가.
- `index.html`: 닫기 버튼을 전용 정리 함수로 연결, PWA 버전 갱신.
- `tests/report-print-layout.test.js`: 조기 정리 금지, 닫기 정리, 1·2면 구성 유지 검증.
- PWA 버전 참조 파일과 `sw.js`: 캐시 버전 동기화.
- `tasks/lessons.md`: 모바일 네이티브 인쇄 미리보기의 `afterprint` 시점과 검증 교훈 추가.

## 검증 계획

- 집중 테스트와 전체 `npm test`.
- `npm run check:en`, esbuild 브라우저 번들, `git diff --check`.
- 실제 A4 PDF가 정확히 1페이지인지 `pdfinfo`로 확인.

## 검증 결과

- `npm test`: 테스트 파일 76개 통과, 1개 건너뜀. 테스트 602개 통과, 에뮬레이터 전용 7개 건너뜀.
- `npm run check:en`: 통과.
- esbuild 브라우저 번들: 통과, 약 1.2 MB.
- `git diff --check`: 오류 없음. Windows 줄바꿈 변환 경고만 존재.
- 실제 PDF: A4 세로 `594.96 × 841.92 pt`, 총 1페이지.
- 렌더 이미지: 상단 첫 면에 제목·요약·식단/운동/마음·포인트 차트가 모두 나타나고, 하단 둘째 면에 카테고리 추이·건강지표·히트맵이 한 번씩 나타남.
- 화면용 원본 모달을 같은 fixture에 함께 두었지만 `A4 한 장에 인쇄`, `닫기`, 화면용 원본 제목은 인쇄되지 않음을 렌더 이미지로 확인.
- 모바일 `390×844`, 데스크톱 `1440×900`: v247 진입점, `print30DayReport()` 및 `close30DayReport()` 연결, 콘솔 오류·경고 없음.
- Android/Samsung 실기기 native 미리보기의 `1/1` 표시는 staging 배포 후 최종 확인이 필요하다.
- 검증용 PDF·PNG·브라우저 프로필과 로컬 서버를 모두 정리했다.
