# 2026-07-13 수동 걸음수 포인트 정책 교정

## 확정된 제품 규칙

- 사용자가 8,000보 이상 입력하면 캡처 이미지나 해시가 없어도 유산소 운동 1회로 인정해 10P를 부여한다.
- 운동 사진 1개와 8,000보 이상 입력이 함께 있으면 유산소 2회로 계산해 15P를 부여한다.
- 8,000보 미만 걸음수는 기록은 저장하지만 운동 포인트 증빙으로 세지 않는다.

## 계획

- [x] 클라이언트와 서버의 걸음 후보 판정을 같은 기준으로 바꾼다.
- [x] 운동 탭 안내 문구를 실제 제품 규칙에 맞춘다.
- [x] 7,999/8,000 경계, 수동 걸음 단독, 걸음+사진 결합을 테스트한다.
- [x] 전체 회귀·번들·Functions 검증을 실행한다.
- [x] 교훈을 갱신하고 커밋·푸시한다.

## 리뷰

- `hasStepPointCredit()`은 캡처 URL·해시와 무관하게 유한한 걸음수가 8,000 이상인지 판정한다.
- 서버 정산은 8,000보를 비미디어 `step_count` 원장 단위로 추가한다. 사진 1개 뒤의 두 번째 유산소 단위이면 5P가 추가되어 합계 15P가 된다.
- `rewardLedgerVersion`을 3으로 올려 기존 v2 기록이 다음 저장에서 새 규칙으로 재평가되도록 했다. 기존 원장 점수는 회수하지 않고 부족한 차액만 추가한다.
- 본서버 배포 후 기존 `adminResettleDailyLogs`를 2026-07-13 범위로 실행하면 아내분을 포함한 오늘 기록이 새 정책으로 즉시 재정산된다.
- 집중 테스트 43개, 전체 Vitest 548개 통과(에뮬레이터 7개 제외). 영문 동기화, mainnet 설정, Functions 문법, esbuild 번들, diff check 통과.
