## 확인 결과

- 기프티쇼 0203 응답의 `code: 0000`은 재발송 요청 API의 정상 접수 응답이며 실제 단말 도달을 증명하지 않는다.
- 공급사 쿠폰 이미지는 `http://` URL로 내려올 수 있어 HTTPS 앱에서 혼합 콘텐츠로 깨질 수 있다. PIN이 있으면 앱 내부 바코드로 대체해야 한다.
- 공급사 상세 응답의 `validPrdEndDt`가 없거나 해석 불가여도 상품 카탈로그의 유효기간을 기준으로 화면과 저장값을 보강해야 한다.

## 안전 원칙

- PIN, 전화번호, 주문번호, 공급사 쿠폰 URL을 작업 문서와 로그 요약에 기록하지 않는다.
- 재발송 확인 과정에서 새 쿠폰 주문이나 포인트 차감이 발생하지 않게 한다.
- 공급사가 요청을 접수했다는 응답을 실제 발송 완료로 안내하지 않는다.

## 리뷰

- 공급사 HTTP 쿠폰 이미지는 HTTPS 화면에서 사용하지 않고 PIN 기반 로컬 바코드로 대체한다. HTTPS 이미지도 로드 실패 시 같은 경로로 전환한다.
- 신규 발급은 유효기간을 항상 저장하고, 기존 누락 기록은 카탈로그의 유효기간과 발급일로 화면에서 복원한다.
- 재발송 성공 응답은 `mmsResendRequestedAt`으로만 기록하며 `deliveredViaMmsAt`을 쓰지 않는다.
