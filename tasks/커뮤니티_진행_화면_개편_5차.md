# 커뮤니티/진행 화면 개편 5차

## 목표
- 갤러리 첫 진입에서 공개 범위를 더 빨리 이해하게 만들기
- 모바일에서 커뮤니티 카드와 갤러리 히어로 높이를 조금 더 줄이기

## 반영 내용
- 갤러리 히어로 설명을 `공유한 기록만 보여요` 중심으로 다시 정리
- 공개 범위 안내 안에 `공유 카드 열기` 버튼을 넣어 행동 흐름을 바로 연결
- 갤러리 피드 설명 문구를 필터별로 더 짧게 축약
- 커뮤니티 카드, 빈 상태, 친구/챌린지 행 간격을 한 단계 더 줄임
- 모바일에서 갤러리 히어로와 피드 패딩을 줄이고 공개 범위 안내 버튼을 세로 정렬로 정리

## 기대 효과
- 갤러리 진입 직후 `기본 비공개` 개념을 더 빠르게 인지할 수 있음
- 공유 행동과 피드 탐색 행동이 서로 덜 헷갈림
- 모바일 첫 화면 높이가 조금 더 낮아져 주요 행동이 빨리 보임

## 검증
- `npm test`
- `node --check`로 `js/app.js` 문법 확인
- localhost 응답 및 핵심 문구 확인
- staging 배포 후 홈 응답 확인

- 피드 필터 칩 우선순위를 실제 사용 흐름 기준으로 재배치할지 검토
