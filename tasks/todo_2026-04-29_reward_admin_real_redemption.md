# 2026-04-29 관제탑 실쿠폰 교환 기준 정리

## 목표
- 관제탑 보상몰 화면도 사용자 실제 쿠폰 교환 기준과 같은 판단 축을 사용한다.
- 비즈머니 운영 하한선은 관제용 참고값으로만 보이고, 사용자 교환 차단처럼 보이지 않게 한다.
- 현재 잔액으로 공개 쿠폰을 몇 건 발급할 수 있는지 관리자에게 바로 보이게 한다.

## 리뷰
- 관제탑 보상몰 요약 카드에서 기존 `가스 예산`, `운영 예산`을 `최저 매입가`, `실발급 가능`으로 바꿨다.
- 공개 쿠폰 2종의 매입가를 기준으로 현재 비즈머니 잔액이 실제로 몇 건 발급 가능한지 계산한다.
- 운영 하한선은 `관제 참고용`, `사용자 차단 아님`으로 표시해 사용자 교환 차단 조건과 분리했다.
- 쿠폰 발급 행에는 상품가와 매입가를 함께 표시해 실제 공급사 비용 흐름을 확인할 수 있게 했다.
- 검증: `npm test`, 앱 번들 esbuild, 관리자 모듈 스크립트 esbuild 파싱 통과.
