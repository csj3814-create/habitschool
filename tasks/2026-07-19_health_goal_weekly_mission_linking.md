# 2026-07-19 기존 주간 미션 중심 건강 실천 흐름 연결

## 목표

새 목표 기능을 만들지 않고 기존 주간 미션을 중심으로 `건강 관심 → 이번 주 작은 실천 선택 → 첫 기록 → 반복 → 기존 30일 결과지` 흐름을 연결한다.

## 작업 원칙

- `users/{uid}.weeklyMissionData`를 건강 목표 실행의 기준으로 사용한다.
- `selectedMissions`는 기존 호환 읽기만 유지한다.
- 새 화면, 컬렉션, 목표 스키마, 이벤트 이름, 보상, AI 기능을 추가하지 않는다.
- 문구, 기존 화면 간 이동, 기본 선택, 표시 순서, 누락 이벤트 호출만 최소 수정한다.
- 기존 작업 트리 변경을 보존하고 커밋·푸시·배포하지 않는다.

## 기존 기능 대응표

| 필요한 역할 | 현재 담당하는 기존 기능 | 재사용 가능 여부 | 최소 변경안 |
|---|---|---|---|
| 건강 목표 선택 | 대시보드 `나의 주간 미션`, `weeklyMissionData` | 가능 | 건강 목표를 이번 주 작은 행동으로 고르는 곳임을 안내하고, 신규 사용자의 기존 `primaryHabit`을 첫 기본 분류로만 활용 |
| 구체적인 행동 결정 | 미션의 `type`, `difficulty`, `target`, 기존 프리셋·직접 추가 | 가능 | 저장 구조는 유지하고 초기 난이도를 `쉬움`으로 설정. 하나만 선택해도 저장 가능하다는 점을 명시 |
| 첫 행동 시작 | 기존 기록 탭과 기록 가이드, 첫 기록 결과 모달 | 가능 | 미션 저장 직후 선택한 분류의 기존 기록 탭으로 이동하고 입력 가이드에 초점 |
| 지속 유도 | 주간 미션 진행률·`missionStreak`, 일일 활동 스트릭, 소모임, 기존 챌린지 | 가능 | 주간 미션 카드에 선택한 미션의 다음 기록 CTA를 연결. 소모임·챌린지·보상 구조는 변경하지 않음 |
| 변화 확인 | 기존 `30일 종합 결과지` | 부분 가능 | 기존 기록 기반 요약 문구만 추가. 주간 미션 이력과 엄밀한 30일 계산은 `추후 검토`로 기록 |
| 보조 보상 | `users.coins`, 포인트 원장, 첫 2,000P 쿠폰, 보상 마켓 | 가능 | 첫 기록의 의미를 먼저 보여준 뒤 기존 포인트·쿠폰 정보를 현재 구조 그대로 표시 |

## 중복·유사 기능 기준

- 주간 목표 기준: `weeklyMissionData`; 레거시 `selectedMissions`는 새 저장에 사용하지 않는다.
- `settings.primaryHabit`: 첫 기록 선호 및 주간 미션 초기 분류 보조이며 별도 목표가 아니다.
- 일일 루틴·마일스톤: 자동 진행·성취 지표이며 목표 선택 기능이 아니다.
- HBT 건강 습관 챌린지: 별도 정산 기능으로 유지하고 이번 진입 흐름에 연결하지 않는다.
- 구 친구 소셜 챌린지: 현재 소모임으로 전환됐으므로 새 연결 대상으로 사용하지 않는다.
- 일일 활동 스트릭·주간 미션 스트릭·리포트 연속 값은 서로 같은 지표처럼 표현하지 않는다.

## 추후 검토 — 이번 범위에서 구현하지 않음

- 현재 결과지는 최신 30개 기록 문서를 읽으므로 엄밀한 달력 30일 참여율이 아니다.
- 현재 결과지의 `최대 연속`은 실제 날짜 간격을 검사하지 않는다.
- 현재 결과지는 `weeklyMissionData`와 `missionHistory`를 읽지 않으므로 선택 미션 실천 이력을 표시하지 않는다.
- 위 항목을 정확히 고치려면 계산·조회 범위가 커지므로 이번 작업에서는 문구만 정확하게 제한한다.

## 리뷰 결과

- 새 화면·컬렉션·사용자 필드·목표 스키마·이벤트 이름 없이 기존 `weeklyMissionData` 중심 흐름만 연결했다.
- `selectedMissions` 호환 읽기, 기존 활성 미션, 포인트·쿠폰 계산, 리포트 조회·차트는 변경하지 않았다.
- 기존 `day3_activated`, `week2_return`만 실제 기록 저장 성공 뒤 판정하며 허용된 버킷 속성만 보낸다. 전송 성공 뒤에만 사용자·이벤트별 로컬 마커를 기록한다.
- `npm test`: 73개 파일, 582개 테스트 통과, 1개 파일 및 에뮬레이터 테스트 7개는 기본 실행에서 의도적으로 제외.
- `npm run test:emulator`: 1개 파일, 7개 테스트 통과. 권한 거부 로그는 실패 경로를 확인하는 예상 출력.
- 최종 집중 테스트: 4개 파일, 46개 테스트 통과.
- esbuild 브라우저 번들 및 `git diff --check` 통과.
- 인앱 브라우저에서 데스크톱 `1440x900`, 모바일 `390x844`의 로그아웃 첫 화면과 비회원 체험을 확인했다. 새 문구 줄바꿈, CTA, 가로 넘침이 없고 일반 CTA의 `dashboard` 의도를 확인했다.
- 인증 사용자 쓰기 흐름은 테스트 계정 자격 증명이 없어 운영 로그인 없이 자동 테스트로 검증했다. 배포 전 테스트 계정으로 온보딩·미션 저장·첫/반복 기록·리포트 기록 상태를 최종 확인해야 한다.
- 커밋·푸시·배포하지 않았다.
