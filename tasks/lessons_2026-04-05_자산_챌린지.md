# 2026-04-05 자산/챌린지 lessons

- 자산 탭의 보조 이력은 새 Firestore 인덱스를 강제하기보다 `daily_logs`와 `blockchain_transactions`를 묶어 보여주고, 정렬은 클라이언트에서 처리하는 쪽이 안전하다.
- 챌린지 예치 슬라이더 상한은 화면에 남아 있는 자산 텍스트를 그대로 믿지 말고, 패널을 열 때 온체인 잔액을 다시 조회해 계산해야 한다.
- 최소 예치금보다 잔액이 적을 때도 슬라이더가 멈춘 것처럼 보이지 않게, `disabled` 상태와 현재 가능한 최대 라벨을 같이 보여줘야 한다.
- 예치 상한 계산에서 `0 HBT`는 미확인 상태가 아니라 실제 잔액으로 취급해야 한다. `> 0` 조건을 쓰면 0 잔액 사용자가 기본 최대치로 잘못 보인다.
