# 커뮤니티/진행 화면 개편 4차

## 목표
- 빈 상태 문구를 더 짧고 바로 이해되게 정리
- 사용자 화면과 관리자 화면의 자산 용어를 통일
- 공개 범위 안내를 커뮤니티 행동 흐름 안으로 연결

## 반영 내용
- 커뮤니티 상단 행동 카드 문구를 `응원하기`, `대화 참여` 중심으로 단순화
- 친구 활동/소셜 챌린지 빈 상태 문구를 짧게 줄이고 CTA를 더 직접적으로 정리
- 커뮤니티 상단에 `기본 비공개` 안내와 `공유하러 가기` 버튼을 추가해 공개 흐름을 연결
- 사용자 자산 탭 용어를 `내 자산`, `현재 포인트`, `현재 HBT` 기준으로 통일
- 관리자 화면 자산 탭과 표 헤더도 `현재 포인트`, `현재 HBT`, `누적 HBT 발행` 기준으로 맞춤

## 기대 효과
- 첫 화면에서 다음 행동을 더 빨리 이해할 수 있음
- 자산 관련 개념을 사용자/관리자 화면에서 같은 단어로 읽을 수 있음
- 기록 공개가 기본 동작이 아니라 선택 동작이라는 점을 커뮤니티 흐름 안에서 바로 인지할 수 있음

## 검증
- `npm test`
- `node --check`로 `js/app.js` 문법 확인
- localhost 응답 확인
- staging 배포 후 홈/관리자 화면 응답 확인

## 다음 작업
- 공개 범위 안내를 갤러리 진입 직전에도 더 짧게 한 번 더 노출할지 검토
- 커뮤니티 카드별 높이를 더 줄일 수 있는지 모바일 기준으로 재확인
