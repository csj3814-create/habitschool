## 결론

지금 구조에서는 해빛코치가 `!공유`를 받았을 때 **해빛스쿨 앱에서 브라우저로만 생성되는 공유 PNG**를 그대로 가져올 수 없다.

이유:
- 해빛스쿨의 공유 이미지는 현재 브라우저에서 `html2canvas`로 즉석 생성된다.
- 즉, 서버에 저장된 “최신 공유 이미지 파일 URL”이 없다.
- 해빛코치는 Firestore는 읽을 수 있지만, 브라우저 메모리에만 있는 PNG는 읽을 수 없다.

따라서 해빛코치 연동은 아래 2가지 중 하나로 해야 한다.

1. 추천 단기안: **해빛코치가 최신 기록을 읽어서 자기 서버에서 공유 이미지를 생성하고 카카오톡으로 보내기**
2. 추천 장기안: **해빛스쿨 서버가 공유 이미지를 canonical하게 생성/저장하고, 해빛코치는 URL만 읽어서 보내기**

이번 문서는 **챗봇 세션에 바로 넘길 수 있는 단기안 중심**으로 적는다.

## 추천 구현안

해빛코치에서 `!공유`를 받으면 아래 흐름으로 처리한다.

1. 카카오 사용자 -> 해빛스쿨 사용자 UID 매핑 확인
2. 해당 UID의 최신 `daily_logs` 조회
3. `shareSettings` 적용
4. 이미지 1~4장 추출
5. 해빛코치 서버에서 공유 카드 PNG 생성
6. 짧은 만료 시간을 가진 공개 URL 발급
7. 카카오 응답으로 이미지 카드 전송

이 방식의 장점:
- 해빛스쿨 앱 추가 배포 없이 챗봇만으로 먼저 붙일 수 있음
- `!공유` 응답 시점에 항상 최신 기록으로 생성 가능
- 카카오에서 바로 이미지 카드로 보여줄 수 있음

주의:
- 앱과 챗봇이 **같은 디자인 코드**를 공유하지 않으므로, 장기적으로는 canonical 서버 렌더링으로 합치는 게 더 좋다.

### 4. 새 파일 `commands/share.js`

새로 만들 파일:

역할:

권장 흐름:

```js
const { getMapping, getDisplayName } = require('../modules/userMapping');
const { getShareCardPayload, createShareCardToken } = require('../modules/appFirebase');

async function handleShare(user) {
  const displayName = getDisplayName(user);
  const mapping = await getMapping(user);
  if (!mapping) {
    return { type: 'text', text: '먼저 !등록 코드 로 계정을 연결해 주세요.' };
  }

  const payload = await getShareCardPayload(mapping.googleUid);
  if (!payload) {
    return { type: 'text', text: '공유할 최신 기록이 아직 없어요. 앱에서 오늘 기록을 저장해 주세요.' };
  }

  const token = await createShareCardToken({
    googleUid: mapping.googleUid,
    kakaoUserKey: user.userId
  });

  return {
    type: 'share-card',
    title: '내 해빛 공유 카드',
    description: '오늘 기록을 카톡으로 바로 보냈어요.',
    imageUrl: `${process.env.RENDER_URL}/api/share-card/${token}.png`,
    webLinkUrl: payload.appUrl || 'https://habitschool-staging.web.app/#gallery'
  };
}
```

### 6. 새 라우트 `GET /api/share-card/:token.png`

수정 파일:
- `C:\SJ\antigravity\habitchatbot\index.js`

새 route 역할:
- 토큰 검증
- 해당 UID 최신 공유 payload 조회
- `renderShareCardPng(payload)` 호출
- `image/png` 응답

예시:

```js
app.get('/api/share-card/:token.png', async (req, res) => {
  try {
    const token = req.params.token;
    const tokenData = await consumeShareCardToken(token);
    if (!tokenData) {
      return res.status(404).send('expired');
    }

    const payload = await getShareCardPayload(tokenData.googleUid);
    if (!payload) {
      return res.status(404).send('no-record');
    }

    const png = await renderShareCardPng(payload);
    res.setHeader('Content-Type', 'image/png');
    res.setHeader('Cache-Control', 'private, max-age=300');
    return res.status(200).send(png);
  } catch (error) {
    console.error('[ShareCard] render error:', error);
    return res.status(500).send('error');
  }
});
```

## 공개 설정 반영 규칙

앱과 맞춰야 하는 규칙:

- `hideIdentity === true`
  - 제목에서 이름 제거
  - `익명 학생의 해빛 루틴` 식으로 대체

- `hideDate === true`
  - 날짜 pill 숨김

- `hidePoints === true`
  - 포인트 pill 숨김

- `hideDiet === true`
  - 식단 미디어 제외

- `hideExercise === true`
  - 운동 미디어 제외

- `hideMind === true`
  - 수면 이미지/감사일기 제외

중요:
- 챗봇이 보내는 공유 카드도 앱과 같은 privacy rule을 써야 한다.
- 즉 `!공유`는 “최신 기록 아무거나”가 아니라 “최신 기록 + shareSettings 적용 결과”를 보내야 한다.

## 챗봇 세션에 바로 넘길 한 문장 요약

해빛코치에서 `!공유`를 추가할 때는 `chatbotLinkCode`로 이미 연결된 `googleUid`를 기준으로 최신 `daily_logs`와 `shareSettings`를 읽고, 챗봇 서버에서 `sharp`로 공유 카드 PNG를 생성한 뒤, `GET /api/share-card/:token.png` 임시 공개 URL을 만들어 `basicCard.thumbnail.imageUrl`로 카카오 응답을 보내면 된다.
