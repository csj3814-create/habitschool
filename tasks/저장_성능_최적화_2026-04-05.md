## 핵심 원인
- 파일 input에 선택된 파일이 저장 후에도 남아 있어서, 다음 저장 때도 새 파일로 판단되어 다시 업로드됐다.
- 운동 영상은 저장 버튼을 눌렀을 때 업로드가 시작돼 체감 지연이 더 컸다.
- 저장 직전에 `daily_logs`를 다시 읽는 시간이 누적돼 텍스트만 바꿔도 저장이 늦게 느껴졌다.

## 반영 결과
- 식단, 유산소 사진, 수면 사진은 선택 즉시 업로드된 결과를 다음 저장 때 그대로 재사용한다.
- 근력 영상도 선택 즉시 업로드를 시작하고, 저장 버튼에서는 이미 올라간 결과를 우선 사용한다.
- 저장이 끝나면 관련 input 값을 비우고 저장된 URL만 DOM/data 속성에 남겨 재업로드를 막는다.
- 오늘 날짜 로그는 메모리 캐시를 우선 사용해 저장 직전 문서 재조회 시간을 줄였다.
