# 2026-07-13 사진·동영상 순차 업로드

## 증상

- 운동 사진 2장 이상을 한꺼번에 선택·저장하면 전송이 동시에 진행되며 `일부 업로드 실패`가 표시된다.
- 같은 위험이 여러 운동 동영상 및 사진·동영상 혼합 저장에도 존재한다.

## 목표

- 사용자는 여러 파일을 한 번에 선택할 수 있다.
- 앱은 선택 순서를 보존해 원본 업로드를 한 건씩 처리하고, 각 파일의 Storage 저장과 daily log 반영을 확인한 뒤 다음 파일로 넘어간다.
- 자동 재시도·오프라인 보관함·썸네일 후처리는 유지하되, 복구 가능한 상태를 사용자에게 실패로 표시하지 않는다.

## 계획

- [x] 현재 pending/background 사진·동영상 전송 및 실패 상태 전이를 추적한다.
- [x] 사진·동영상 공용 순차 업로드 큐를 설계하고 저장 흐름에 연결한다.
- [x] 큐가 선택 순서, 실패 후 다음 항목 진행, 자동 재시도, 백그라운드 patch를 보존하도록 구현한다.
- [x] 다중 사진·다중 동영상·혼합 업로드 회귀 테스트를 추가한다.
- [x] 전체 테스트, 영문 동기화, 번들, PWA 캐시 검증을 통과시킨다.
- [ ] 커밋·푸시 후 스테이징에서 실제 순차 동작을 확인하고 배포 범위를 보고한다.

## 리뷰

- 운동 사진·동영상 입력에 다중 선택을 허용하고, 선택된 파일을 파일당 하나의 운동 카드로 분리한다.
- 원본, 이미지/동영상 썸네일, 백그라운드 재시도를 하나의 FIFO Storage 큐로 직렬화한다.
- 사전 업로드가 실패해도 파일을 유지해 저장 시 다시 시도하며, 복구 가능한 상태에는 빨간 실패 UI를 표시하지 않는다.
- PWA 캐시를 v237로 갱신했다.
- 검증: 집중 테스트 48개 및 전체 Vitest 538개 통과, 영문 동기화·메인넷 설정·Functions 문법·esbuild 번들 통과.
