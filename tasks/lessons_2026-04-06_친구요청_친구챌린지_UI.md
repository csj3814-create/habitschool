## 2026-04-06 친구 요청/친구 챌린지 UI 교훈

- 친구 요청 상태는 서버 응답만 기다리지 말고, 성공 직후 화면에서 `신청중`으로 즉시 바뀌는 낙관적 상태를 같이 넣는다.
- 앱 안에서 친구 요청을 처리하는 경로가 여러 개라면 갤러리, 프로필, 챌린지 영역이 모두 같은 `friendships` 원장을 보도록 유지한다.
- 기능을 숨기거나 섹션을 재배치할 때는 렌더 함수만 남기지 말고 실제 HTML 마운트 지점도 같이 확인한다.
- 초대/친구 연결 UI가 여러 탭에 있으면 기능은 나눠도 시각 구조와 카피 톤은 같은 컴포넌트 규칙으로 맞춘다.
- 친구 요청 관련 새 흐름을 넣을 때는 `보낸 요청`, `받은 요청`, `활성 친구`, `친구 챌린지 노출`까지 한 번에 QA한다.
