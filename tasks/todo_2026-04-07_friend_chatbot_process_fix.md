## 리뷰
- 친구 연결은 서버가 이미 `active` 관계를 반환해도 초대 코드 경로가 이를 UI 상태에 반영하지 못했고, 앱은 `users/{uid}.friends` 캐시를 무시한 채 `friendships` 쿼리만 기다리다가 타임아웃 시 "친구를 먼저 연결해요"로 되돌아가고 있었다.
- `loadMyFriendships()`가 사용자 문서의 `friends` 배열을 먼저 활성 친구 캐시로 시드하고, 이후 `friendships` 실데이터로 덮어쓰도록 바꿔서 느린 모바일 환경과 일시적 쿼리 실패에서도 친구 상태가 유지되게 했다.
- 초대 코드/직접 추가 경로 모두에서 `already_friends`와 `pending_exists`를 분기 처리해 토스트 문구와 실제 UI 캐시가 일치하도록 맞췄다.
- 해빛코치 연결은 보류 토큰이 남아 있을 때 일시적 네트워크 오류도 앱 진입마다 즉시 재시도하면서 같은 실패 토스트를 반복하던 것이 원인이었다.
- 실패 토큰은 별도 로컬 상태로 기록하고 자동 재시도에는 쿨다운을 두었으며, 프로필 카드 안에서 사용자가 직접 `다시 확인` 또는 `보류 정리`를 선택할 수 있게 바꿨다.
- 검증: `npm test` 통과, `npx esbuild js/app.js --bundle --format=esm --platform=browser --outfile=$env:TEMP\\habitschool-app-check.js` 통과.
