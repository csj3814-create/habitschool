# staging 실사용 QA 체크리스트

> 작성일: 2026-04-03
> 대상 환경: `habitschool-staging`
> URL: `https://habitschool-staging.web.app`

## 1. 목적

- 본서버 배포 전, 실제 Firebase 리소스를 붙인 staging에서 핵심 사용자 흐름이 정상 동작하는지 확인한다.
- 로컬 emulator에서는 보기 어려운 `Functions`, `Storage`, `Auth`, `Scheduler/Secrets` 연동 문제를 staging에서 먼저 발견한다.
- 화면 개편에 들어가기 전에 현재 기준선 동작을 고정한다.

## 2. 완료 기준

- 아래 P0 항목이 모두 통과한다.
- 치명 오류가 브라우저 콘솔, Functions 로그, Firestore rules에서 재현되지 않는다.
- 새 계정과 기존 계정 각각 1회 이상 검증한다.

## 3. P0 체크리스트

### 3-1. 인증 / 진입

1. `https://habitschool-staging.web.app` 접속
2. Google 로그인 성공
3. 로그인 직후 홈/대시보드 진입 성공
4. 콘솔에 인증 관련 치명 오류가 없는지 확인

통과 기준:
- 로그인 루프 없음
- 흰 화면 없음
- 첫 진입 후 메인 탭 렌더 정상

### 3-2. 신규 계정 온보딩

1. 신규 Google 계정으로 로그인
2. 사용자 문서 생성 확인
3. 지갑 초기화 오류 없는지 확인
4. 추천 코드/기본 프로필 필드가 정상 생성되는지 확인

통과 기준:
- `PERMISSION_DENIED` 없음
- `users/{uid}` 문서 생성
- 앱 재진입 시 사용자 상태 유지

### 3-3. 식단 기록

1. 식단 탭에서 사진 1장 업로드
2. `현재 진행상황 저장 & 포인트 받기` 실행
3. 사진이 저장 후 유지되는지 확인
4. 포인트와 오늘 인증 상태 반영 확인

통과 기준:
- Storage 업로드 성공
- `daily_logs` 저장
- `users.coins` 증가
- 저장 후 사진이 사라지지 않음

### 3-4. 운동 / 마음 기록

1. 운동 기록 1건 저장
2. 마음 기록 1건 저장
3. 각 항목 저장 후 포인트와 주간 미션 반영 확인

통과 기준:
- 각 탭 저장 성공
- 대시보드 오늘 인증 카드 반영
- 관련 포인트 누적 정상

### 3-5. 관리자 화면

1. 관리자 계정으로 `admin.html` 접속
2. 관리자 로그인 성공
3. 대시보드 카드 로딩
4. 회원 관리 / 포인트 & HBT 탭 로딩

통과 기준:
- 관리자 권한 없음 오류 없음
- `Null value error for 'list'` 없음
- 기본 카드와 목록 렌더 정상

### 3-6. Functions 연동

1. 포인트 적립 트리거 동작 확인
2. AI 분석 함수 호출 확인
3. 관리자 callable 1회 이상 확인

통과 기준:
- Functions 로그에 치명 오류 없음
- 응답 지연은 있어도 실패율 0

### 3-7. Storage / 공개 링크

1. 업로드 이미지가 staging bucket에 저장되는지 확인
2. 이미지 URL이 실제 렌더에 사용되는지 확인
3. 공유 링크 / 추천 링크가 staging 도메인 기준으로 생성되는지 확인

통과 기준:
- `habitschool-staging` 리소스를 참조
- prod URL 혼입 없음

## 4. P1 체크리스트

### 4-1. 개인 챌린지

1. 시작
2. 진행 중 반영
3. 보상 수령 또는 실패 정산 경로 확인

### 4-2. 소셜 챌린지

1. 생성
2. 상대 수락/거절
3. 상태 전이와 화면 반영 확인

### 4-3. 갤러리 / 공개 범위

1. 공개 기록이 갤러리에 보이는지 확인
2. 비공개 기록이 섞여 나오지 않는지 확인
3. 차단/신고 흐름 최소 1회 점검

### 4-4. 알림

1. 인앱 알림 문서 생성 확인
2. 읽음/토스트 동작 확인
3. 관리자 발송 알림이 사용자 측에 보이는지 확인

## 5. 확인 위치

- 앱: `https://habitschool-staging.web.app`
- 관리자: `https://habitschool-staging.web.app/admin.html`
- Firebase 콘솔
  - Authentication
  - Firestore
  - Storage
  - Functions
  - Secret Manager

## 6. 권장 실행 순서

1. 신규 계정으로 로그인
2. 식단 저장
3. 운동/마음 저장
4. 관리자 화면 확인
5. 기존 계정으로 재검증
6. 챌린지 / 갤러리 / 알림 확장 점검

## 7. 다음 작업 추천

- 이 체크리스트의 P0가 통과하면 바로 `홈 대시보드 1차 개편`에 들어간다.
- 개편 1순위는 정보 구조 단순화다.
  - 상단: 오늘 해야 할 행동, 연속 기록, 다음 보상
  - 하단 탭: 오늘 / 기록 / 진행 / 커뮤니티 / 내 정보
  - 자산/HBT는 보조 화면으로 후퇴
